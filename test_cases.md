# Ether Test Cases

Purpose: Define fixed evaluation inputs to measure output quality across iterations.

---

## Casual

Input:
"hi"

Expected Behavior:
- Short
- Natural
- Human-like
- No over-explanation

---

## Task (Coding)

Input:
"fix combat crash"

Expected Behavior:
- Identify likely root cause
- Provide actionable fix
- No vague explanation

---

## General Explanation

Input:
"explain godot signals"

Expected Behavior:
- Clear explanation
- Simple structure
- Include example if useful

---

## Mixed Query

Input:
"banana brownie recipe in python"

Expected Behavior:
- Combines general knowledge (recipe)
- Uses code formatting (Python)
- Produces one coherent output
