# Software Design Patterns
Updated: 2026-04-22

## Creational Patterns

### Singleton
Ensure only one instance exists.
Use case: Game managers, configuration

### Factory
Create objects without specifying exact class.
Use case: Enemy spawning, item generation

### Builder
Construct complex objects step by step.
Use case: Level generation, UI construction

## Structural Patterns

### Observer
One-to-many dependency notification.
Use case: Event systems, achievements

### Component
Compose objects from reusable parts.
Use case: ECS architecture, character customization

### Adapter
Make incompatible interfaces work together.
Use case: Legacy code integration

## Behavioral Patterns

### State
Object changes behavior based on internal state.
Use case: Game states, character animations

### Strategy
Interchangeable algorithms at runtime.
Use case: AI behaviors, combat systems

### Command
Encapsulate requests as objects.
Use case: Undo/redo, input queuing
