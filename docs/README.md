# Ether Documentation Structure

This repository keeps **active, user-facing docs** close to the code and removes stale phase-by-phase status dumps from the root.

## Primary docs

- `README.md` — project overview, installation, architecture, and usage.
- `README_CLI.md` — CLI-focused workflow and examples.
- `tests/README.md` — testing conventions and command usage.

## Why this structure

- Reduces noise in the repository root.
- Keeps documentation focused on maintained, actionable guides.
- Avoids drift from outdated milestone reports.
